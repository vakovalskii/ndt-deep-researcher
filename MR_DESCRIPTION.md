# Fix: Исправление проблем запуска проекта

## Описание проблемы
При попытке запуска проекта пользователи сталкивались с ошибкой:
```
ValueError: LLM API Key is required
```
несмотря на правильную настройку переменных окружения в `.env` файле.

## Корневая причина
1. **Неправильные имена переменных**: В README указывались `LLM_API_KEY`, `LLM_API_BASE`, `LOCAL_LLM`, но код ожидал `OPENAI_API_KEY`, `OPENAI_API_BASE`, `OPENAI_MODEL`
2. **Проблемы загрузки .env**: Переменные окружения не загружались автоматически при создании объекта Configuration
3. **Отсутствие python-dotenv**: Не было автоматической загрузки .env файлов

## Внесённые изменения

### 1. Исправления в `src/assistant/configuration.py`
- ✅ Добавлена автоматическая загрузка `.env` файлов через `python-dotenv`
- ✅ Логика чтения переменных окружения перенесена в `__post_init__`
- ✅ Гарантированная загрузка переменных при любом способе создания объекта
- ✅ Убраны отладочные print'ы для production кода

### 2. Улучшенная документация
- ✅ **PROJECT_OVERVIEW.md** - краткий обзор архитектуры проекта
- ✅ **SETUP_GUIDE.md** - подробное руководство по установке и troubleshooting
- ✅ **Обновлён README.md** с правильными именами переменных

### 3. Правильные переменные окружения
```env
# Было (неправильно)
LLM_API_KEY=...
LLM_API_BASE=...
LOCAL_LLM=...

# Стало (правильно)
OPENAI_API_KEY=...
OPENAI_API_BASE=...
OPENAI_MODEL=...
```

## Инструкции для тестирования MR

1. Создайте `.env` с правильными переменными:
```env
TAVILY_API_KEY=your_key
OPENAI_API_KEY=your_key
OPENAI_API_BASE=https://your-server/v1/chat/completions
OPENAI_MODEL=your_model
```

2. Запустите проект:
```bash
source .venv/bin/activate
set -a && source .env && set +a
langgraph dev
```

3. Убедитесь, что нет ошибки "LLM API Key is required"
